What This Project Does

iridium-sniffer is a standalone C-based Iridium burst detector and demodulator. It reads live IQ from SDR hardware or IQ files, detects bursts, demodulates them, and emits iridium-toolkit compatible RAW output. Optional features add ACARS/SBD decoding (with text position extraction and waypoint lookup), a live web map with CSV paging export, Mission Control GUI, GSMTAP output for Wireshark, ZMQ integration, VITA 49 input, BaseStation SBS feed for ADS-B trackers, and Doppler-based receiver positioning.

10 MHz
Default sample rate for full authorized Iridium coverage
1622 MHz
Default center frequency for wideband receivers
16 dB
Default burst threshold for maximum recovery
5050 / 8888
Mission Control default port / web map default port

Operating Modes

                        
Each mode adds capability, but each also adds moving parts. Start with RAW output, then layer features one at a time.

                    

                


                

                    

                        
1. Base decode

                        
Use this to validate RF chain, gain, and burst detection before adding visualizers or higher-layer decoders.

                        
./iridium-sniffer -i soapy-0

                    


                    

                        
2. Web map

                        
Starts the built-in HTTP server and shows ring alerts, satellites, aircraft, and receiver position when available.

                        
./iridium-sniffer -i soapy-0 --web

                    


                    

                        
3. ACARS / SBD

                        
Decodes IDA-carried aviation and short burst data directly in the binary. No Python pipeline required. Extracts aircraft positions from ADS-C structured data, ACARS free text coordinates (three format variants), and waypoint database lookups. Now also accepts uplink IDA frames (ft=6/7) in addition to downlink.

                        
./iridium-sniffer -i soapy-0 --acars

                    


                    

                        
4. Wireshark

                        
Sends decoded IDA frames to GSMTAP/LAPDm over UDP for Wireshark inspection.

                        
./iridium-sniffer -i soapy-0 --gsmtap

                    


                    

                        
5. Position solving

                        
Uses Doppler shift from multiple satellites to estimate receiver position. Best with stable clocks and open sky.

                        
./iridium-sniffer -i soapy-0 --position

                    


                    

                        
6. Parsed output

                        
Emits parsed IDA lines to stdout for reassembler workflows. Non-IDA frames remain as RAW lines.

                        
./iridium-sniffer -i soapy-0 --parsed

                    


                    

                        
7. BaseStation SBS feed

                        
Outputs decoded aircraft positions in MSG,3 BaseStation format on a TCP server (default port 30003). VRS, tar1090, PlanePlotter, and other ADS-B tracking tools can connect and receive Iridium-derived aircraft positions alongside ADS-B data. Aircraft registrations are mapped to real ICAO hex addresses using a 568K-entry tar1090-db aircraft database. Groundspeed and heading are derived from consecutive position fixes.

                        
./iridium-sniffer -i soapy-0 --acars --basestation

                        

                            4-tier position priority
                            ICAO hex mapping
                            GS + heading
                        

                    


                    

                        
8. CSV paging export

                        
The web map now accumulates paging events (TMSI, satellite, beam, position, timestamp) in the browser and provides an Export Pages CSV button on the dashboard bar for offline analysis and AIS correlation.

                        
./iridium-sniffer -i soapy-0 --web --acars

                    

                

            


            

                

                    Practical rule:
                    
Low ok% does not automatically mean poor performance. This project defaults to a more aggressive 16 dB threshold than gr-iridium, so it detects more weak bursts. That can lower the success percentage while still increasing total decoded frame count.

                

            


            

                

                    

                        
Mission Control

                        
The GUI is a browser-based control surface for launching the sniffer, watching live output, viewing spectrum and constellation plots, and rendering the 3D globe. In this repo it is started by the PowerShell launcher at the project root.

                    

                


                

                    

                        
1

                        
Launch from Windows

                        
Run the provided launcher. It starts the Flask GUI in WSL, prepares a Python virtual environment if needed, and opens your browser.

                        
./Start-MissionControl.ps1
# or
./Start-MissionControl.ps1 -Port 8080

                    


                    

                        
2

                        
Confirm the GUI tabs

                        
Expect tabs for RAW Output, System Log, Spectrum, Constellation, Decoded Data, and 3D Globe.

                    


                    

                        
3

                        
Start simple

                        
Begin with a base live capture profile, confirm decoded bursts, then enable globe, web map, ACARS, or position solving only after base data flow is clean.

                    

                


                

                    

                        
Mission Control is best for

                        
    
                            
  • Monitoring stdout and stderr without terminal juggling
    • 
                            
  • Watching constellation quality via the DQPSK scatter plot
    • 
                            
  • Using the 3D Globe without manually wiring satellite visualization pieces
    • 
                            
  • Controlling sessions from a browser while the backend stays in WSL
    • 
                        

                    


                    

                        
Mission Control is not a substitute for

                        
    
                            
  • Proper SDR gain staging
    • 
                            
  • Correct sample rate and center frequency selection
    • 
                            
  • Antenna placement and sky visibility
    • 
                            
  • Verifying raw CLI behavior before adding GUI complexity
    • 
                        

                    

                

            


            

                

                    

                        
Field Reference

                        
These are the settings and outputs that operators most often need to interpret during real sessions.

                    

                


                

                    

                        
                            

                                Item
                                Default
                                Meaning
                                When to change it
                            

                        
                        
                            

                                Center frequency
                                1622000000
                                Wideband default covering the authorized Iridium band with a 10 MHz sample rate.
                                Shift to 1625500000 for narrowband SDRs such as RTL-SDR targeting ring alerts.
                            

                            

                                Sample rate
                                10000000
                                Balanced default for broad coverage and expected pipeline tuning.
                                Use 2400000 for narrowband devices or 12000000 for extended coverage experiments.
                            

                            

                                Threshold
                                16 dB
                                Detection sensitivity. Lower catches more weak bursts, higher reduces noise-triggered work.
                                Raise toward 18 dB when you want cleaner ok% stats or lower CPU churn from weak detections.
                            

                            

                                Web map port
                                8888
                                HTTP server for the built-in map.
                                Change if the port is already in use or you want multiple sessions.
                            

                            

                                Mission Control port
                                5050
                                Flask GUI port used by the PowerShell launcher.
                                Change when running another local service on 5050.
                            

                            

                                RAW output
                                stdout
                                iridium-toolkit compatible demodulated frames.
                                Switch to ZMQ PUB when you want multiple consumers.
                            

                            

                                BaseStation port
                                30003
                                TCP server for SBS MSG,3 aircraft position output. tar1090, VRS, and PlanePlotter connect here.
                                Change with --basestation=PORT or push to a remote host with --basestation=HOST:PORT.
                            

                            

                                --basestation-beam
                                off
                                When enabled, beam-estimated positions (~200 km accuracy) are also sent via the SBS feed. Default sends only GPS-quality positions (ADS-C, free text, waypoint).
                                Enable when you want approximate coverage in tar1090 and accept reduced accuracy.
                            

                            

                                Aircraft database
                                ~/.iridium-sniffer/aircraft.csv
                                Maps aircraft registrations to ICAO hex addresses (568K entries from tar1090-db).
                                Use --aircraft-db=PATH for a custom location. Run --update-db to download/update.
                            

                            

                                Waypoint database
                                data/waypoints.csv
                                125K-entry aviation waypoint database for fallback position resolution when ACARS messages reference waypoints by name.
                                Bundled with the build. No user action needed unless replacing with a custom dataset.
                            

                        
                    

                


                

                    

                        
Healthy first signs

                        
    
                            
  • Device listing works and selected SDR opens without backend errors
    • 
                            
  • stderr shows steady stats instead of repeated startup failures
    • 
                            
  • RAW lines appear with real frequencies inside the Iridium band
    • 
                            
  • Web map or Mission Control updates every second when enabled
    • 
                        

                    


                    

                        
Likely setup issues

                        
    
                            
  • No bursts at all: gain, antenna, band coverage, or wrong device selection
    • 
                            
  • Many bursts, low useful decode: threshold too low, poor SNR, unstable clock, or weak front-end settings
    • 
                            
  • Position solver not converging: insufficient satellite diversity, obstructed sky, or unstable timing
    • 
                            
  • Mission Control fails early: WSL, Python, or GUI venv dependencies not available
    • 
                        

                    

                

            


            

                

                    

                        
Troubleshooting

                        
Use the search box to narrow the list. Search terms such as gain, WSL, map, ACARS, Wireshark, or RTL work well.

                    

                    
                


                

                    My SDR does not appear in the device list
                    

                        
    
                            
  • Run the binary with the device listing mode first rather than guessing an interface string.
    • 
                            
  • Confirm the matching backend library is installed and was detected at build time.
    • 
                            
  • For SoapySDR hardware, use deterministic device args if multiple devices are attached.
    • 
                            
  • On Windows GUI workflows, remember that the launcher runs the GUI in WSL. Your SDR access path must make sense from that environment.
    • 
                        

                    

                


                

                    I see bursts but decode quality is poor
                    

                        
    
                            
  • Raise threshold toward 18 dB if the detector is spending time on too many weak marginal bursts.
    • 
                            
  • Adjust device-specific gain rather than assuming one aggregate gain value fits all hardware.
    • 
                            
  • Check antenna, LNA, filters, and coax losses before blaming demodulation.
    • 
                            
  • Use the constellation or scatter plots in Mission Control to see whether symbol clusters are stable.
    • 
                        

                    

                


                

                    The web map opens but shows no useful activity
                    

                        
    
                            
  • Confirm the base sniffer is actually decoding before diagnosing the map layer.
    • 
                            
  • The map depends on decoded IRA and IBC-related data. If your capture bandwidth misses the relevant channels, visible activity will be limited.
    • 
                            
  • For narrowband devices, use the ring-alert centered profile around 1625500000 Hz and 2400000 samples per second.
    • 
                            
  • Make sure you are loading the correct local port if you changed it with --web=PORT.
    • 
                        

                    

                


                

                    Mission Control does not launch correctly
                    

                        
    
                            
  • The PowerShell launcher requires WSL to exist and the gui folder to be reachable from the WSL mount path.
    • 
                            
  • On first run, the launcher creates a Python virtual environment and installs gui requirements. If Python venv creation fails in WSL, fix that first.
    • 
                            
  • If the browser opens but the page is unavailable, verify the selected port is not blocked or already occupied.
    • 
                            
  • Run the GUI manually from the gui folder in WSL if you need to isolate launcher issues from Flask issues.
    • 
                        

                    

                


                

                    Wireshark shows no GSMTAP packets
                    

                        
    
                            
  • Start Wireshark on the correct local interface and filter UDP port 4729 unless you changed the destination.
    • 
                            
  • GSMTAP depends on IDA decode output, so if no IDA frames are being recovered there will be no Wireshark traffic.
    • 
                            
  • If you redirected the host or port with --gsmtap=HOST:PORT, validate that the destination is reachable.
    • 
                        

                    

                


                

                    ACARS feed output does not match what I expect
                    

                        
    
                            
  • Use --acars for human-readable terminal output, --acars-json for dumpvdl2-style JSON, and --feed for aggregator-targeted iridium-toolkit JSON.
    • 
                            
  • libacars is optional. Without it, basic ACARS extraction works but deeper ARINC-622 decoding is limited.
    • 
                            
  • Check station identifiers and endpoint syntax when feeding external collectors.
    • 
                        

                    

                


                

                    BaseStation SBS feed is not showing aircraft in tar1090/VRS
                    

                        
    
                            
  • Confirm --basestation is enabled and your tracker is connecting to the correct port (default 30003).
    • 
                            
  • The SBS feed requires --acars to be active since positions come from ACARS/SBD decode.
    • 
                            
  • By default only GPS-quality positions are sent (ADS-C, free text coordinates, waypoint lookups). Use --basestation-beam to also include beam estimates.
    • 
                            
  • Run --update-db once to download the aircraft database for ICAO hex address mapping.
    • 
                            
  • If positions appear but with wrong coordinates, verify MSG,3 field count is 22 (this was fixed in a recent update).
    • 
                        

                    

                


                

                    Position source priority and quality
                    

                        
    
                            
  • Aircraft positions follow a 4-tier priority: (1) ADS-C GPS structured data (highest confidence), (2) free text lat/lon from ACARS message body, (3) waypoint database lookup, (4) beam cache estimate (~200 km accuracy).
    • 
                            
  • Free text extraction supports three coordinate formats: POS[NS]ddddd[EW]dddddd, [NS]DDmmm[EW]DDDmmm, and LAT [NS] dd.ddd/LON [EW] ddd.ddd.
    • 
                            
  • Groundspeed and heading are computed from consecutive GPS-quality fixes separated by at least 10 seconds, with sanity checks (0.5–700 knots).
    • 
                            
  • Beam estimates remain visible on the built-in web map but are excluded from the BaseStation SBS feed unless --basestation-beam is set.
    •

Workflow Checklists

                        
These short sequences are the fastest way to avoid self-inflicted complexity.

                    

                


                

                    

                        
Checklist: First live capture

                        
    
                            
  • List devices and select the exact interface string you intend to use.
    • 
                            
  • Run base decode with no extra features.
    • 
                            
  • Tune gain and threshold until bursts and useful RAW lines are stable.
    • 
                            
  • Add web map or Mission Control after the CLI is already healthy.
    • 
                        

                    


                    

                        
Checklist: Narrowband RTL-SDR setup

                        
    
                            
  • Use a ring-alert centered profile around 1625500000 Hz.
    • 
                            
  • Use a 2400000 sample rate and expect narrower coverage than wideband hardware.
    • 
                            
  • Prioritize web map and ring-alert activity rather than full-band ambitions.
    • 
                        

                    


                    

                        
Checklist: Doppler positioning

                        
    
                            
  • Start with clean decode and open sky.
    • 
                            
  • Prefer stable clocks and known altitude if available.
    • 
                            
  • Be patient: convergence needs enough measurements from multiple satellites.
    • 
                        

                    


                    

                        
Checklist: Feeding external tools

                        
    
                            
  • Use RAW stdout or ZMQ when driving iridium-toolkit components.
    • 
                            
  • Use --feed for aggregator endpoints such as airframes.io or acarshub.
    • 
                            
  • Use --gsmtap when the destination is Wireshark rather than a text or JSON consumer.
    • 
                            
  • Use --basestation to feed tar1090, VRS, or PlanePlotter with Iridium-derived aircraft positions.
    • 
                        

                    


                    

                        
Checklist: BaseStation SBS setup

                        
    
                            
  • Enable both --acars and --basestation flags.
    • 
                            
  • Run --update-db once to download the aircraft database for ICAO hex mapping.
    • 
                            
  • Point tar1090/VRS at localhost:30003 (or your custom port).
    • 
                            
  • Optionally add --basestation-beam if you want approximate beam positions in the feed.
    • 
                            
  • Verify positions appear by checking the tracker's aircraft list for Iridium-sourced entries.
    •
